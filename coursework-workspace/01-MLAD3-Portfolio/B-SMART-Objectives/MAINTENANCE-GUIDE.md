## Making Changes to Objectives

### When to Create Change Log Entry

**Always create a Change Log entry when:**
- Changing target numbers (e.g., 70 → 50 hours/week)
- Shifting deadlines (e.g., Nov 30 → Dec 15)
- Modifying scope (e.g., adding/removing features)
- Changing approach (e.g., manual → automated)
- Adjusting priorities (e.g., High → Critical)

**No Change Log needed for:**
- Routine progress updates
- Adding evidence links
- Completing milestones
- Typo fixes

### Change Log Entry Template

```
Date: [YYYY-MM-DD]
Objective: [Objective Name]
Change Type: [Major / Minor / Correction]
Description: [What changed?]
Rationale: [Why did it change?]
Impact: [High / Medium / Low]
Author: [Your name]
```

### Example Change Log Entry

```
Date: 2025-10-04
Objective: Objective 6: Partnership App + 100 Agencies
Change Type: Major
Description: Reduced target from 10,000 to 1,000 end clients (100 agencies × 10 instead of 100 each)
Rationale: AI model feedback identified unrealistic projections. 100 clients per agency unachievable in 4-month timeline with zero current partnerships.
Impact: High
Author: Shaan
```

## Feedback Response Process

### 1. Receive Feedback

- Team member provides feedback during review
- AI analysis provides feedback
- Self-reflection identifies issue

### 2. Document Feedback

- Create entry in Feedback database
- Include: Date, Reviewer, Type, Full text
- Mark Status: "New"

### 3. Evaluate Feedback

Ask yourself:
- Is this feedback valid? (Yes/No/Maybe)
- Does it align with marking criteria?
- Does it strengthen my portfolio?
- Is it feasible to implement?

### 4. Decide Response

- **Accept:** Implement change, create Change Log entry
- **Challenge:** Document why you disagree, discuss with reviewer
- **Partial:** Implement portion, document rationale for rest

### 5. Act on Feedback

- Make changes to objective if accepted
- Create Change Log entry
- Update Feedback entry with Response
- Mark Status: "In Progress" or "Addressed"

### 6. Verify Impact

- Did the change improve the objective?
- Is it aligned with Learning Contract?
- Document outcome in objective page

## Updating Progress Ratings

### How to Calculate Progress %

**Linear Progress Model:**
```
Progress % = (Current Value - Baseline) / (Target - Baseline) × 100
```

**Example (Objective 1):**
- Baseline: 30 hours/week
- Current: 40 hours/week
- Target: 70 hours/week
- Progress = (40 - 30) / (70 - 30) × 100 = 25%

**Binary Progress Model (for on/off milestones):**
```
Progress % = (Completed Milestones / Total Milestones) × 100
```

**Example (Objective 5):**
- Total milestones: 10
- Completed: 4
- Progress = 4/10 × 100 = 40%

### When to Update Progress

**Update Weekly:**
- Objectives with active work
- Metrics that changed significantly (>5%)
- After completing milestones

**Keep As-Is:**
- Objectives not started (0%)
- Objectives blocked (current %)
- Objectives waiting for dependency (current %)

## Milestone Tracking

### Creating Milestones

**Good Milestones:**
- Specific and measurable
- Have clear deadline
- Can be verified complete/incomplete
- Progress objective toward goal

**Bad Milestones:**
- Vague ("work on app")
- No deadline
- Can't verify completion
- Doesn't advance objective

### Milestone Status Values

| Status | Icon | When to Use |
|--------|------|-------------|
| Complete | ✅ | 100% done, verified |
| In Progress | ⏳ | Started, not finished |
| Pending | ⏳ | Not started, scheduled |
| Blocked | 🚫 | Cannot start, dependency not met |
| Cancelled | ❌ | No longer relevant |

**Documents:**

- Specifications

## Troubleshooting

### Issue: Progress Not Updating

**Check:**
- Are you tracking the right metric?
- Is baseline/target still accurate?
- Any external factors affecting progress?

**Solution:**
- Recalculate using correct formula
- Update baseline/target if needed
- Document rationale in Change Log

### Issue: Milestone Slipping

**Check:**
- Is deadline realistic?
- Any blocking dependencies?
- Resource constraints?

**Solution:**
- Adjust deadline (document in Change Log)
- Remove milestone if no longer relevant
- Break into smaller milestones

### Issue: Objective Blocked

**Check:**
- Which dependency is blocking?
- Can dependency be accelerated?
- Can objective be resequenced?

**Solution:**
- Update status to "Blocked"
- Document blocking issue
- Create unblocking plan
- Update timeline if needed

### Issue: Too Many Objectives At Risk

**Check:**
- Are priorities correct?
- Are timelines realistic?
- Resource allocation issue?

**Solution:**
- Monthly maintenance: reassess all objectives
- Deprioritize lower-importance items
- Request team feedback on rebalancing
