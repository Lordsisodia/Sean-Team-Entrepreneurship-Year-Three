## üöÄ Quick Start Guide

### 1. Understand the System
Read these first:
- [.docs/VERSIONING-GUIDE.md](.docs/VERSIONING-GUIDE.md) - How versioning works
- [.docs/SECTION-MAPPING.md](.docs/SECTION-MAPPING.md) - How folders map to questions

### 2. Work on a Section
Navigate to any section folder and read its `README.md`:
```bash
cd 01-where-have-i-been/
cat README.md
```

### 3. Review Current Content
Each section has a `versions/vXX/draft.md` file:
```bash
cat 01-where-have-i-been/versions/v01/draft.md
```

### 4. Iterate Using Versioning
When you receive feedback:
1. Document it in `versions/v01/feedback.md`
2. Create `versions/v02/` and copy the draft
3. Apply changes in the new draft
4. Update `metadata.json`

## üìù Versioning Quick Reference

### Creating a New Version
```bash
# 1. Create new version folder
mkdir sections/01-where-have-i-been/versions/v02

# 2. Copy and revise draft
cp versions/v01/draft.md versions/v02/draft.md
# Edit versions/v02/draft.md

# 3. Create empty feedback file
touch versions/v02/feedback.md

# 4. Update metadata
# Edit versions/v02/metadata.json
```

### Version Status Values
- `draft` - Initial version, not yet reviewed
- `under-review` - Awaiting feedback
- `revising` - Incorporating feedback
- `complete` - Final version, no changes needed
